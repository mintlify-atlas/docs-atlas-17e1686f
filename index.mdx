Meteor is a lightweight, plugin-driven agent for collecting metadata. It compiles into a single binary with no external dependencies, and connects to databases, dashboards, messaging systems, and cloud services to gather metadata and forward it to any data catalog or downstream system.

## How Meteor works

Meteor uses **recipes** — YAML files that define where to extract metadata from, how to process it, and where to send it. Each recipe specifies one source (extractor), optional processors, and one or more sinks.

<Steps>
  <Step title="Install Meteor">
    Install the Meteor binary on your machine using Homebrew or download a release directly.

    ```bash
    brew install raystack/tap/meteor
    ```
  </Step>
  <Step title="Create a recipe">
    Generate a sample recipe for your data source using the `meteor new` command.

    ```bash
    meteor new recipe my-recipe -e bigquery -s console
    ```
  </Step>
  <Step title="Run the recipe">
    Execute the recipe to extract and sink metadata.

    ```bash
    meteor run my-recipe.yaml
    ```
  </Step>
</Steps>
